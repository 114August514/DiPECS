## 当前实现确认

会议确认 DiPECS 当前是 Android/Linux AIOS 原型，主链路为：

```text
Android collector JSONL / daemon system sources / replay fixture
    -> aios-collector / RustCollectorIngress
    -> IngestedRawEvent
    -> PrivacyAirGap
    -> SanitizedEvent
    -> WindowAggregator
    -> StructuredContext
    -> DecisionRouter
    -> IntentBatch
    -> PolicyEngine + CapabilityLevel
    -> ActionLifecycle
    -> AuthorizedAction
    -> ActionAdapter
    -> AuditRecord / runtime trace / replay audit hash
```

核心边界：

- `RawEvent` 只存在于 collector 到 `PrivacyAirGap` 的短路径。
- 模型后端只接收 `StructuredContext`。
- 决策后端只输出 `IntentBatch`。
- `AuthorizedAction` 只能由 `ActionLifecycle` 构造。
- 每个动作坐标恰好产出一条终态 `AuditRecord`。

## 已完成能力

### Android 采集

- `UsageStatsManager` promoted。
- `NotificationListenerService` promoted。
- `DeviceContext` heartbeat promoted。
- `AccessibilityService` 保留 screening，不进入生产 Rust schema。
- Android append-only JSONL 可由 daemon tail 进入 Rust 管线。

### 决策与策略

- `RuleBasedBackend` 默认路径。
- `LocalEvaluatorBackend` 可用于本地评估。
- `CloudLlmBackend` 支持 DeepSeek、Qwen/DashScope 和 OpenAI-compatible endpoint。
- `FallbackNoOpBackend` 用于失败和熔断。
- `PolicyEngine` 覆盖风险、置信度、能力、urgency、target-in-context 和 allow-list。

### 动作治理

- `ActionLifecycle` 已成为 `AuthorizedAction` 唯一 seal 点。
- `DefaultActionExecutor` 支持本地 stub。
- `AndroidAdapter` 可在 bridge 配置完整时转发 Android-safe action。
- Android socket 使用 token、freshness window、HMAC、payload 限制、读超时和失败退避。

### Replay 与审计

- `aios-cli replay` 支持 ingest、sanitize、context、decision、policy、execute 等 stage。
- `OfflineAdapter` 用于 deterministic replay。
- canonical audit stream 和 `audit_hash` 用于 golden regression。
- daemon 支持 runtime trace 输出。

## 已知缺口

会议确认以下内容不在最终主线完成范围内：

- 真机 Android 长期验证尚未完成。
- Android APK 构建依赖本机 SDK 或 CI 环境。
- Binder/eBPF 只有接口和检测 stub，当前不产生真实 Binder 事件。
- fanotify/system image 仍是后续路线。
- `PreWarmProcess`、`KeepAlive`、`ReleaseMemory` 在 Android 上语义已收缩为自有资源动作或用户可见提示，不是系统级强动作。
- 历史文档中关于 Skills 自迭代和 MockCloudProxy 的表述应视为阶段性方案。

## 最终交付口径

最终展示应强调：

1. DiPECS 不是让 LLM 直接控制系统，而是让模型在本地安全框架内参与决策。
2. 项目的核心贡献是隐私隔离、机制策略分离、授权动作执行和确定性审计。
3. Android public API production ingress 已经打通，系统可通过 JSONL/replay/daemon 路径验证。
4. 对未完成部分要诚实说明，尤其是真机验证和系统级采集。
