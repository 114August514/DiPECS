## 背景

动作桥接从本地 stub 走向 Android localhost socket 后，安全风险变高。会议认为 bridge 必须遵守两个原则：

- Rust 侧只能转发 `AuthorizedAction`。
- Android 侧只执行公开 API 范围内的安全子集。

## 讨论内容

### 1. Socket 鉴权和 HMAC

Android `AuthorizedActionSocketServer` 监听 `127.0.0.1:46321`。

会议确认：

- ping payload 需要 `auth_token`。
- token 存储在 Android `EncryptedSharedPreferences`。
- 执行动作使用 `message_type: "execute"` envelope。
- envelope 包含 `issued_at_ms` 和 `expires_at_ms` freshness window。
- `auth.hmac_sha256` 覆盖 freshness window 和 length-prefixed serialized `AuthorizedAction`。

这样可以降低本地跨应用注入、重放和畸形 payload 风险。

### 2. Payload 和连接限制

Android socket 需要增加防护：

- payload 最大 64 KiB。
- 读超时。
- auth 失败退避。
- 最大 client 线程限制。
- pending client 限制。
- Android 返回 JSON status，Rust 只把 `status: "ok"` 映射为 forwarded。

CLI 的 `send-authorized-action` 保持 health-check ping，不直接派发动作。

### 3. Prefetch 网络限制

`PrefetchFile(url:...)` 需要防止 SSRF 和本地网络访问。

会议确认限制：

- 只允许 `https://`。
- 拒绝 localhost。
- 拒绝 private、link-local、multicast、unique-local IPv6。
- redirect 目标必须重新校验。
- 最大下载 2 MiB。
- cache 默认 24 小时 TTL。

### 4. 隐私字段修复

会议检查 Android notification interaction 事件，认为 Android notification key 可能携带 tag 或应用侧识别信息，不应进入模型或 audit。

处理方向：

- notification key 在脱敏阶段丢弃。
- 模型输入只保留 `TextHint`、`SemanticHint` 和来源类别等结构化摘要。
- provider 错误不记录 HTTP 响应体。

## 决议

- Android bridge 必须使用 HMAC execute envelope。
- Android-safe action target 使用前缀白名单。
- `PrefetchFile` 增加网络目标限制。
- notification interaction 中可能携带 PII 的 key 不跨越 Privacy Air-Gap。
- 期末周期间优先修安全漏洞和文档，不安排大规模新功能。
