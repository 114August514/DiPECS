## 讨论重点

本次会议把“AIOS”进一步拆成两个可能方向。

第一类是资源隔离与调度，例如 Agent Cgroup、Action-Aware Co-scheduler。这类方向更接近传统 OS 研究，但需要找到足够明确的 workload 和评测指标。

第二类是面向用户意图的系统辅助。AutoGLM 等项目说明大模型可以在用户界面层理解和执行任务，但如果直接使用屏幕或 UI 文本作为接口，会带来隐私、可控性和安全问题。这个观察后来影响了 DiPECS 的设计：模型不能直接读原始数据，也不能直接执行动作。

## 初步结论

大家倾向继续保留“用户行为感知 + 系统优化”的方向，把它和 AIOS 结合起来，而不是单纯做应用层 Agent。
