## 报告要点

### 系统闭环

"本地采集 → 本地整理脱敏 → 云端大模型调用 skills 判断 → 本地低风险优化 → skills 自迭代"

### 技术栈

| 层级 | 语言 | 职责 |
| ---- | ---- | ---- |
| Android 应用层 | Kotlin | 权限、服务、UI、上下文采集、优化动作调用 |
| 核心逻辑层 | Rust | 事件模型、上下文聚合、脱敏、策略校验、日志回放 |
| 云端判断层 | Remote LLM + Skills | 场景理解、skill 选择与调用、置信度判断 |

### 四阶段进度

1. Android 端基础采集
2. Rust 核心处理链路
3. 打通云端大模型与 skills 调用
4. 本地优化与反馈记录

### 主要风险

- **权限和隐私**：脱敏 + 降级运行 + 用户知情
- **网络与云端服务**：超时降级 + 缓存 + 非阻塞
- **skills 自迭代**：只更新非高风险参数 + 版本保留 + 观察模式
- **系统复杂性**：Rust 集中核心逻辑 + 阶段目标清晰 + 云端不直接控制设备
